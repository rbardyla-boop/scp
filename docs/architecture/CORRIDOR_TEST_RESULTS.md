# Corridor Reality Test — Results

**Version**: 1 — FROZEN for five-pair collection run.
Do not revise this document's scoring schema, aggregate table structure, or gate criteria
until all five pairs are complete and scored. Record results as collected.
If vocabulary is revised after the run, open a Version 2 results file for re-test sessions.

---

**Protocol**: See `ARCHITECTURE_REALITY_GATE.md` §2 and `CORRIDOR_FACILITATOR_SCRIPT.md`.
**Materials**: `CORRIDOR_PARTICIPANT_SHEET.md`, `CORRIDOR_SCORING_RUBRIC.md`.

**Gate threshold**: ≥ 80% Correct per state/term across all participants,
AND zero unresolved critical misinterpretations per term.
**Minimum sample**: 5 bilateral pairs (10 participants). Fewer does not satisfy the gate.
**Ambiguous counts as Not Correct for threshold purposes.** Record it separately for vocabulary revision guidance.
**Critical-misinterpretation veto**: A term fails even at 80%+ Correct if any participant produces
a response that could cause unsafe trust behaviour and the confusion is plausibly caused by the label.
The veto overrides the percentage. See `CORRIDOR_SCORING_RUBRIC.md` Part A for the specific
veto trigger for each term.

**Overall status**: `PENDING — not yet executed.`

**Instrument version**: 1.2 — wireframe-based (no live SCP runtime required).
Participants see static ASCII mock screens; no device setup or clock manipulation is performed.

**Interpretation limit** (record before first session, do not fix before this run):
Each participant sees both the A and B establishment mock screens. This test therefore
measures comprehension of the complete presented interaction flow, not what a user would
infer from only one device's local screen. This is acceptable for this first formative gate.
A later client usability test should assign participants to actual endpoint roles to isolate
single-endpoint inference. Do not treat this run's findings as evidence about single-endpoint
screen comprehension.
